# 实验2

## 实验目标

学习一个 **structure-aware real-only latent completion prior**，并分别测量
 **结构背离、细节背离、结构-细节耦合失配**，同时继续保留 mask-conditioned instability 作为关键异常信号。

我们不仅要考虑补出来的内容偏不偏，细节补的稳不稳，细节是不是依附在正确的结构上。因为AIGC 图像极其擅长伪造局部中高频的统计连续性，但往往缺乏宏观物理空间的因果逻辑。

## 架构

### 编码器E

选择ConvNeXt系Encoder，从零开始做real-only训练（第二选择是小型hybrid encoder，前面卷积stem，后面少量transformer block，不作为首发主实验）

通过编码器我们将图像尽可能完整压缩到潜空间中

同时，我们采用DC-AE的思想，让Latent空间结构化解耦

我们将结构通道Z_s放在前32个通道，强制其负责编码宏观结构，几何边缘和全局空间因果逻辑

细节通道Z_d放在后96个通道，负责存储高频纹理、光影细节和局部噪点分布

##### 输入

256 * 256 * 3 的像素

##### 输出

16 * 16 * 128 （或者其他，主要看模型特点）的特征图Z，也就是我们说的潜变量

### 2D正弦-余弦位置编码

将16 * 16的特征图展平为256的1D序列后，注入2D坐标。对于网格中的任意坐标（x,y），分别对其计算64维的正弦/余弦嵌入，拼接成128维的位置向量，直接加到对应的Token上。

### 掩码模块 f

采用Masked AutoEncoder的思想，但我们在 latent 网格上做掩码，而不是在像素空间做掩码。主实验对比两类 mask：

- `random token mask`
- `block mask`

被 mask 的 token 替换成一个可学习 `[MASK]` 向量，同时带有2D位置编码。

训练和测试都要做 **mask ratio sweep**，测试重点报告 ` / 40% / 50% / 60% / 70% / 75%`。
 不要预设 75% 一定合理，因为不同 mask 形状下难度完全不同。

这样最后不只是得到一个点，而是得到一条曲线。
 这条曲线本身就是结果。

### 补全网络 P

使用2到4层标准的Transformer Encoder（仅包含Multi-Head Self-Attention和FFN）。依靠携带2D位置编码的剩余 真实 Token，在自注意力机制中推理并填补那  mask掉 的部分，输出补全后的完整潜空间 z_hat

输入是带 2D 位置编码的 masked latent，输出是补全后的完整 latent $\hat Z$。

### 轻型解码器D

使用中等容量的轻型解码器（如转置卷积或 PixelShuffle + Conv，或少量转置卷积）。

原则只有两个：
 一是足够表达阶段一的 latent-to-pixel 映射；二是不能强到在阶段二替 P 做大规模结构修复

在stage1阶段，解码器D单独挂两个小辅助头

```
H_s(Z_s) -> X_low
H_d(Z_d) -> X_detail_target
```

`H_s` 和 `H_d` 是**以 `Z_s` / `Z_d` 为输入的独立辅助头**

这样主解码器 D 负责 full recon，结构/细节代理用各自的小head。

## 数据与预处理

### 训练数据

阶段一和阶段二都只用真实图像训练，数据源是 **DIV2K + Flickr2K**。训练时统一裁成 `256×256` patch

### 预处理规范

real 和 fake 在测试时必须走同一条 pipeline，包括：

- 相同解码方式
- 相同 resize / crop 策略
- 相同颜色空间
- 相同归一化
- 尽可能统一文件格式链路

这样做是为了尽量排除格式和图像链路偏差。

## 训练流程

### 一阶段训练：real-only AE 预训练

#### 目标

学习一个稳定的 real latent dictionary，即 `X -> E -> Z -> D -> \hat X`。
 这一阶段只训练 `E` 和 `D`，不启用 `P` 和 `f`

#### 数据增强策略

实验二的的初步过程中我们先不加任何数据增强

#### 损失函数

$$\mathcal L_{\text{stage1-new}}
=
\mathcal L_{\text{full-recon}}
+
\lambda_s \mathcal L_{\text{struct}}
+
\lambda_d \mathcal L_{\text{detail}}$$

其中L_full-recon就是我们实验一中的pixel L1和freq loss。在此基础上我们增加L_struct监督只用前32个通道去解码一个结构代理目标。

##### L_full-recon

$$\mathcal L_{\text{full-recon}}=\mathcal L_{\text{pixel-L1}}+\lambda_f\mathcal L_{\text{freq}}$$

##### L_struct

对原图做高斯模糊或低通。得到X_low，然后要求仅用Z_structj = Z[: , :32]重建X_low

注意这里用的D的辅助头H_s

$$L_{\text{struct}} = \lambda_{s1} L_1 + \lambda_{s2} L_{\cos}$$

（第二种方案是从原图提取HOG descriptor map或Sobel/Canny边缘图，然后要求前32通道预测这些结构代理）

##### L_detail

$$L_{\text{detail}} = L_1
\quad \text{或} \quad
\text{Huber/Charbonnier}$$

让后96个通道负责重建细节残差

注意这里用的D的辅助头H_d

我们detail head最后一层不要sigmoid，直接输出实值，用L1/L2去拟合残差

（第二种方案是预测梯度图/Laplacian图）

#### 结束条件

阶段一收敛后，**冻结 D**。
 主实验里建议 **E 也冻结到阶段二**，这样第二阶段的目标最干净。后续“E 小学习率微调”只作为消融，不放进主流程。

### 二阶段训练：masked latent completion prior 训练

第二阶段主实验固定 `E` 和 `D`，**只训练 P**。
 这是本方案里最关键的收口点。这样可以避免 latent drift 干扰解释

激活的模块：f（Mask率要好好斟酌）和P（配合2D位置编码）

冻结的模块：D，E。

#### 数据流

1. `X -> E -> Z`
2. `Z -> f -> Z_masked`
3. `Z_masked -> P -> \hat Z`
4. 在 latent 空间计算 stage2 loss

> `D` 仅用于阶段一训练或阶段二可视化分析，不参与第二阶段主损失计算。

#### 训练策略

主实验建议：

- 训练的mask ratio采样自一个小集合，主体分布为：40%，60%，75%，辅助分布为5%，10%。其中主体分布主导梯度，学习空间因果推演；辅助分布防止分布偏移，学习近乎全可见状态下的保守补全。网络 $P$ 输出推演特征 $Z_{\text{pred}}$ 后，在送入解码器或计算 Loss 前，强制执行物理替换：$$\hat{Z} = M \odot Z_{\text{pred}} + (1-M)\odot Z$$，以此确保所有可见区域（Visible Tokens）的流形被 $100\%$ 原样保留

- mask 类型：`random` 和 `block` 混合采样，建议各占一半

这样可以避免 P 只学会局部插值。

#### 第二阶段损失

第二阶段Loss写成分组非对称形式：

$$\mathcal L_{\text{stage2}}
=
\lambda_s \mathcal L_{\text{mask-struct}}
+
\lambda_d \mathcal L_{\text{mask-detail}}$$

其中：

- `mask-latent-L1`：只在被 mask 的 token 上计算。约束补全结果在数值上贴近真实 latent
- `mask-latent-cos`：只在被 mask 的 token 上计算。降低纯能量差异的影响，更关注方向一致性
- λ_s = 1.0
- λ_d = 0.08

##### L_mask-struct

$$\mathcal L_{\text{mask-struct}}
=
\alpha_1 \mathcal L_1^{(s)}
+
\alpha_2 \mathcal L_{\cos}^{(s)}$$

##### L_mask-detail

$$\mathcal L_{\text{mask-detail}}
=
\beta_1 \mathcal L_1^{(d)}
+
\beta_2 \mathcal L_{\cos}^{(d)}$$

spatial mask 作用于被选中 token 的**全部通道**，只是训练和评估时再按 `Z_s/Z_d` 分组统计。

首版主实验**不把 masked pixel loss 放进核心损失**。
 因为 latent token 到 pixel 区域并不是严格一一对应，硬切 patch 很容易把 decoder 感受野边界误差也算进去。
 若后续要做 masked pixel loss，只能在 pixel 端对 mask 做膨胀映射后再算。

另外，这里我们把loss和score分通道统计，也就是说分别算

- `structure channels` 上的 masked loss
- `detail channels` 上的 masked loss

## Zero-Shot 鉴别

#### 推理流程

对每张待测图像 `X`：

1. 先提取一次 `Z = E(X)`
2. 对 `Z` 做 `K` 次不同 mask
3. 每次都经 `P` 得到 `\hat Z_k`
4. 比较 `\hat Z_k` 与原始 `Z` 在 mask 区域上的差异
5. 聚合得到最终异常分数 

#### K 的默认设置

首发建议：

- `K = 8` 作为默认
- 再做 `K = 4 / 8 / 16` 稳定性分析

因为 K 太小不稳，太大则推理线性变慢。

#### 多重随机探测

1. 对待测图像X，提取特征Z
2. 对其进行 K 次不同的 独立采样，分别经过 P次补全得到 $\hat{Z}_k$

#### 最终评分设计

##### 结构背离分数

只在前32通道上计算

$$S_{\text{struct}}
=
\log
\frac{
\mathbb E[\mathrm{Err}_{mask}^{\text{struct}}] + \tau_s
}{
\mathbb E[\mathrm{Err}_{base}^{\text{struct}}] + \tau_s
}$$

测试宏观结构prior是否被违反

##### 细节背离分数

只在后96通道上计算

$$S_{\text{detail}}
=
\log
\frac{
\mathbb E[\mathrm{Err}_{mask}^{\text{detail}}] + \tau_d
}{
\mathbb E[\mathrm{Err}_{base}^{\text{detail}}] + \tau_d
}$$

##### 结构-细节耦合失配分数

$$S_{\text{couple-lite}}
=
\log
\frac{
\mathrm{Var}_{\text{detail}} + \tau_d
}{
\mathrm{Var}_{\text{struct}} + \tau_s
}$$

$\mathrm{Var}$ 测量的是网络在 $K$ 次不同 Mask 下的**认知不确定性**

**对于真实图像**：它的结构和细节是物理统一的。当给出部分上下文时，网络对结构的推演很笃定（$\mathrm{Var}_{\text{struct}}$ 小），由于结构定下来了，依附在结构上的自然纹理走向也就大体确定了（$\mathrm{Var}_{\text{detail}}$ 也相对较小）。因此，这个比率维持在一个稳定的基准区间。

**对于 AIGC 图像（降维打击时刻）**：生成图像的结构和细节是强行缝合的。网络在推演时，或许能勉强根据边缘猜出结构（$\mathrm{Var}_{\text{struct}}$ 保持在尚可的水平），当网络试图把自然界的细节附着在这个伪造的结构上时，会发现无论怎么补都和周围那极其平滑的生成纹理发生剧烈冲突。导致网络对细节的预测在 $K$ 次探测中\极度发散、疯狂乱跳（$\mathrm{Var}_{\text{detail}}$ 呈指数级爆炸）。

分子剧烈膨胀，分母相对稳定，这个 $S_{\text{couple-lite}}$ 分数将成为判别 AIGC “皮肉分离（结构细节失配）”的指标。

##### 辅助分数

```
S_var_struct
S_var_detail
```

##### 分析分数

###### 方差分数

将全通道总方差分成S_var_struct, S_var_detail，当然还是要算全通道总方差S_var，然后再加一个S_var_cross方差，就是说结构稳定的时候细节是否仍然乱跳

> 其中 `\tau` 是基于真实验证集 base 误差统计得到的 floor，不是单纯的机器 epsilon。
>

> 这个很好，而且有个优点：
>
> - 当两者相等时，分数正好是 0
> - 当 mask 误差更大时，分数为正
> - 数值上比较平滑
>
> `\tau` 取真实验证集上 `Err_base` 的 5% 分位数，作为数据驱动的稳定地板。

###### base 的严格定义

**测试时的 `Err_base`**：统一使用 **`5% mask`**，并且**必须与评测的 mask_type 严格对齐**（即评估 Block 掩码时，Base 使用 Block 5%；评估 Random 掩码时，Base 使用 Random 5%）。对 2-4 次不同低 mask 采样取平均。相应的 $\tau_s$ 和 $\tau_d$ 也必须按 mask_type 独立统计。

$$S_{\text{var}} = \mathrm{Var}_k(\hat Z_k)$$

对每个 token iii，只统计那些“该 token 在本次前向中确实被 mask”的 runs 上的方差，然后再在 token 维度求平均。

`S_var` 必须在**固定评估条件**下计算，不能把不同 mask ratio 或不同 mask type 的 runs 混在一起。



## 评估报告项

#### 核心报告项

你最终至少要报告下面几组结果：

1. `mask ratio sweep` 曲线：/40/50/60/70/75

2. `random mask` 与 `block mask` 的对比

3. `K=4/8/16` 的稳定性

4. 主分数：

   - `S_struct`
   - `S_detail`
   - `S_couple-lite`

   辅助分数：

   - `S_var_struct`
   - `S_var_detail`

   分析分数：

   - `S_var`
   - `S_var_cross`

   以及这些分数的AUROC，ACC(阈值固定)，AP(Average Precision )

> 对每个评估条件，单独计算 `S_struct`、`S_detail`、`S_couple-lite` 及其辅助分数。
>

这样最后得到的是一组曲线，而不是一个混合分数。

#### complexity 相关性分析

因为这个方法天然可能混入“局部可预测性”而不只是“真假差异”，所以必须额外做相关性分析。
 测试时记录 score 与以下量的相关性：

- 边缘密度
- 高频能量
- patch entropy
- 梯度能量 

如果发现异常分数与这些复杂度指标高度相关，就说明你的 score 里混入了 complexity signal。这不是坏事，但必须明确诊断。

#### Sanity Check

验证前32/后96通道分工是否真实成立

- 仅用前32通道重建 `X_low`
- 仅用后96通道重建 `X_detail`
- 用前32通道去重建 `X_detail`，应显著更差
- 用后96通道去重建 `X_low`，应显著更差
