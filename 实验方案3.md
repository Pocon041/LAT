# 实验三

## 实验目标

在真实图像上训练一个像素域的Masked image prior。如果一张测试图像符合自然图像的分布，那么被遮挡的patch更容易被真实先验恢复；如果是AIGC图像，网络在遮挡率大的情况下会出现更高的 Masked Reconstruction Divergence 和 更高的多次补全不稳定性。

## 数据和预处理

训练数据使用 DIV2K + Flickr2K

输入统一 为 256 * 256

reak/fake 测试时走同一条pipline：相同解码，resize/crop、颜色空间、归一化、文件格式链路

## 模型结构

输入：256 * 256 * 3

patchify： 16 * 16 patch size.因此单张图像被切成 `16 × 16 = 256` 个 patch。

encoder： ViT-Small

decoder：轻量transformer decoder，2-4层即可

position embedding：2D sin-cos，延续实验1，2

实验三采用标准 MAE 范式：

1. 在 patch 网格上生成 mask
2. encoder 仅处理 visible patches
3. decoder 接收 visible latent tokens 与 mask tokens
4. decoder 对完整 patch 序列输出预测
5. 训练时仅在 masked patches 上计算重建损失

## Mask设计

#### Mask类型

主训练使用：random mask，block mask。比例1：1

stripe / half-region mask只在测试扩展使用

#### Block mask定义

在 patch 网格上随机采样 block 的长宽，使其面积接近目标 mask ratio。
 宽高比限制在 `[1/2, 2]`，避免出现极端狭长形状主导训练。

### Mask ratio

训练时，mask ratio 采用混合采样：

- 主体分布：`40% / 60% / 75%`
- 辅助分布：`5% / 10%`

其中主体分布主导梯度，学习在中高遮挡条件下的补全 prior；辅助分布用于防止分布偏移，使模型在接近全可见状态下也保持稳定。

测试时做 mask ratio sweep：

- `40% / 50% / 60% / 70% / 75%`

不要预设高遮挡一定更好。对不同 mask type，最终结果以整条曲线为准，而不是单一比例点。



### 评分区域：masked core

实验三的训练损失和测试评分区域**必须分开定义**。

#### 训练时

训练损失统一在 **full masked region** 上计算，不做 erosion。

#### 测试时

为了降低 patch 边界误差和感受野过渡误差对分数的污染，评分只在 **masked core** 上进行。定义如下：

1. 先在 patch 网格上采样 mask
2. 再映射到 pixel mask
3. 对 pixel mask 做一次 `8 px` 宽度的 erosion
4. erosion 后剩余区域定义为 `masked core`

所有测试阶段的 `S_raw-pixel`、`S_high-freq` 与 `S_pixel-var`，均只在 `masked core` 上统计。

#### visible区域处理

训练与推理前向都遵循原生 MAE 机制：
 visible patches 通过 encoder，并与 mask tokens 一起进入 decoder，decoder 输出完整 patch 序列预测。

但为了获得稳定的检测分数，在**测试阶段**对 decoder 输出做 visible copy-back 合成：

$$\hat X = M \odot X_{\text{pred}} + (1-M)\odot X$$

其中：

- masked 区域使用 decoder 的预测
- visible 区域强制替换为原图

后续所有评分、可视化和高频误差统计，均基于该合成后的 `\hat X` 进行。
 这样做的目的是避免 decoder 在 visible 区域引入无意义漂移，从而污染像素误差、频域误差和预测方差。

## 训练损失

训练损失只在 **full masked region** 上计算，不在 `masked core` 上算。

$$\mathcal L_{\text{Exp3-main}}
=
\lambda_1 \mathcal L_{\text{masked-raw-L1}}^{\text{full-mask}}
+
\lambda_2 \mathcal L_{\text{masked-lap-L1}}^{\text{full-mask}}$$

其中：

- `masked-raw-L1`：

$$\mathcal L_{\text{masked-raw-L1}}^{\text{full-mask}}
=
\frac{1}{|M|}
\sum_{u \in M}
\left|
\hat X(u) - X(u)
\right|$$

即在 **full masked region** 上计算 raw RGB 的 L1 损失。

- `masked-grad`：Laplacian

$$\mathcal L_{\text{masked-lap-L1}}^{\text{full-mask}}
=
\frac{1}{|M|}
\sum_{u \in M}
\left|
\Delta \hat X(u) - \Delta X(u)
\right|$$

其中 Δ\DeltaΔ 表示拉普拉斯算子（Laplacian operator），用于提取图像的二阶空间导数与高频边缘细节。

主协议中，高频项的唯一实现为 **Laplacian L1**。



## 推理与打分

对每张待测图像 `X`：

1. 采样一个固定评测条件：`mask type + mask ratio`
2. 做 `K` 次独立 mask
3. 每次得到一张补全结果 `\hat X_k`
4. 对每次结果执行 visible copy-back
5. 仅在 `masked core` 区域上计算误差或方差
6. 聚合为最终异常分数

默认设置：

- `K = 8`
- 额外报告 `K = 4 / 8 / 16` 稳定性分析



### 实验三真正该打的三个分数

#### 原生像素误差分数

$$S_{\text{raw-pixel}}
=
\log
\frac{
\mathbb E_k[\mathrm{Err}^{\text{raw}}_{\text{mask-core}}] + \tau_r
}{
\mathbb E_j[\mathrm{Err}^{\text{raw},(j)}_{\text{base}}] + \tau_r
}$$

其中 `Err_mask^raw` 就是 **masked core region** 上的 raw pixel L1。

$$\mathrm{Err}^{\text{raw}}_{\text{mask-core}}
=
\frac{1}{|\Omega_{\text{core}}|}
\sum_{u \in \Omega_{\text{core}}}
\left|
\hat X(u) - X(u)
\right|$$ 

这里$$\Omega_{\text{core}}$$表示 `masked core` 像素集合。

衡量在较大遮挡条件下，像素域补全误差相对于图像自身低遮挡基准水平膨胀了多少。

#### 高频异常分数

$$S_{\text{high-freq}}
=
\log
\frac{
\mathbb E_k[\mathrm{Err}^{\text{hf}}_{\text{mask-core}}] + \tau_h
}{
\mathbb E_j[\mathrm{Err}^{\text{hf},(j)}_{\text{base}}] + \tau_h
}$$

其中主实现为：

$$\mathrm{Err}^{\text{hf}}_{\text{mask-core}}
=
\frac{1}{|\Omega_{\text{core}}|}
\sum_{u \in \Omega_{\text{core}}}
\left|
\Delta \hat X(u) - \Delta X(u)
\right|$$

即仅在 `masked core` 上统计 **Laplacian L1**。
 辅助实现可额外报告 `windowed FFT high-band amplitude divergence`，但不进入主分数。

物理意义：
 衡量 AIGC 图像在底层高频、边缘和局部纹理统计上的异常程度。

#### 像素预测方差分数

$$S_{\text{pixel-var}}
=
\mathrm{Var}_k(\hat X_k)$$

定义上只统计那些：

- 在该次前向中确实被 mask 的位置
- 且属于 `masked core` 的像素

然后再在空间维度求平均。

要求：

- `S_pixel-var` 必须在固定 `mask type`、固定 `mask ratio` 下计算
- 不能混不同 mask 类型
- 不能混不同遮挡比例
- 只基于 copy-back 后的 `\hat X_k` 统计

物理意义：
 衡量像素域补全过程在多次独立 mask 探测下是否稳定。真实图像通常呈现较低方差；AIGC 图像更容易在局部像素与纹理预测上发生不一致。



## Err_base 与 τ\tauτ 的严格定义

由于实验三的评分仅在经过 `8 px erosion` 后的 `masked core` 上进行，极低比例的稀疏 random mask 可能导致 `masked core` 有效像素数过小甚至为 0。为避免 base 校准误差失效，实验三采用以下**统一且严格**的 base 协议：

#### 1. Base 误差

- 当评估 `block mask` 条件时，`Err_base` 使用 **10% block mask**，并对 `2–4` 次独立采样取平均。

- 当评估 `random mask` 条件时，`Err_base` 使用 **10% random mask**，并要求

  masked_core_pixel_count≥Nmin⁡\text{masked\_core\_pixel\_count} \ge N_{\min}masked_core_pixel_count≥Nmin

  否则该次采样作废并重采样。

这里定义：

Nmin⁡=256N_{\min} = 256Nmin=256

即 base 采样后，`masked core` 的有效像素数至少要达到 256 个像素，否则视为无效样本。

#### 2. Floor 常数

- τr\tau_rτr：真实验证集上 `Err_base^raw` 的 `5%` 分位数
- τh\tau_hτh：真实验证集上 `Err_base^hf` 的 `5%` 分位数

两者都必须按**相同 base 协议**分别统计。

#### 3. 方差分数的有效性约束

`S_pixel-var` 同样只在有效 `masked core` 上统计。
 若某次采样满足不了 `N_{\min}` 约束，则该次采样作废并重采样。

## Complexity 相关性分析

因为 pixel-space MAE 也可能混入“局部可预测性”与“图像复杂度”信号，所以必须做额外相关性分析。测试时记录分数与以下量的相关性：

- 边缘密度
- 高频能量
- patch entropy
- 梯度能量

如果发现异常分数与这些复杂度指标高度相关，则说明 score 中混入了 complexity signal。这个结果不是坏事，但必须被明确诊断和报告。

## 基线

实验三至少对比以下基线：

1. **Vanilla real-only AE reconstruction error**
2. **Trivial inpainting baseline**：zero fill / mean fill / local interpolation
3. **实验一：latent completion prior**
4. **实验二：structure-aware latent prior**

这样才能回答两个问题：

- pixel-domain masked prior 本身是否有效
- 它与 latent-space prior 相比处于什么位置

## 消融实验

实验三至少做以下消融：

1. `random mask` vs `block mask`
2. `with self-calibration` vs `without`
3. `with visible copy-back` vs `without`
4. `raw L1 only` vs `raw L1 + Laplacian`
5. `40/60/75 + 5/10` vs `60/75/85 + 5/10`

第五项只作为扩展消融，不纳入主协议。
